# Laravel Lesson レビュー①

## Todo一覧機能

### Todoモデルのallメソッドで実行しているSQLは何か
SELECT * FROM `todos'  
  
### Todoモデルのallメソッドの返り値は何か
Eloquentのコレクション  
  
### 配列の代わりにCollectionクラスを使用するメリットは
・一貫性と可読性  
コレクションを使用することで、データ操作のコードが一貫しており、可読性が向上する。  
また、複雑なループや条件分岐を避け、シンプルなメソッドチェーンで表現できる。  
  
・豊富なメソッド  
コレクションは、データのフィルタリング、変換、集計など、さまざまな操作を簡単に行うための豊富なメソッドを提供し、  
これにより、開発者は効率的にデータを操作できる。  
  
・オブジェクト指向  
コレクションはオブジェクト指向の考え方に基づいており、データとその操作を一つのオブジェクトとして扱う。  
これにより、コードが直感的で保守しやすくなる。  
  
### view関数の第1・第2引数の指定と何をしているか
第一引数  
laravel内のviewsフォルダ内にあるファイルを検索して表示させる。  
  
第二引数  
ビューに渡すデータ（連想配列形式）  
  
役割としてはコントローラからビューにデータを渡し、そのビューをブラウザに表示すること。  
  
### index.blade.phpの$todos・$todoに代入されているものは何か
 controllerのコードにある  
 public function index()  
{  
    $todo = new Todo();  
    $todos = $todo->all();  

    return view('todo.index', ['todos' => $todos]);  
}  
  
→return view('todo.index', ['todos' => $todos]);  
この行で、  
ビュー名 → 'todo.index'  
ビューに渡すデータ → ['todos' => $todos]  
が指定されている。  
  
$todos = $todo->all();  
のコードは、SQLで言うところの、  
SELECT * FROM todos;  
を意味するので、つまり、todos は、todosテーブルの全レコード を取得した結果。  
  
## Todo作成機能
  
### Requestクラスのallメソッドは何をしているか
HTTPリクエスト（フォーム送信など）で送られてきた全データを連想配列として取り出す処理をしている。  
  
### fillメソッドは何をしているか
Eloquentモデル における「データ一括代入（mass assignment）」のための便利な機能。フォームから送信されたデータを 一括でモデルのプロパティに代入 するために使われる。  
  
### $fillableは何のために設定しているか
どのカラムに一括代入（mass assignment）を許可するかを指定するための設定。  
もし $fillable を指定しないで  
ユーザーの送信データ（$request->all()）をそのまま代入すると、  
悪意あるユーザーが 意図しないカラム に値を送ってしまう可能性がある。  
  
### saveメソッドで実行しているSQLは何か
オブジェクトの状態をDBに保存するINSERT文を実行している。  
  
INSERT INTO `todos` (`content`, `updated_at`, `created_at`)   
VALUES ('焼肉', '2025-10-13 12:00:00', '2025-10-13 12:00:00');  
  
### redirect()->route()は何をしているか
別のページ（ルート）へ移動させる」ための関数。  
つまり、ユーザーを指定した画面へ リダイレクト（再読み込み転送）をしている。  
  
## その他
  
### テーブル構成をマイグレーションファイルで管理するメリット
SQLを知らなくても、PHPコードでテーブル操作ができるため学習コストが不要。  
マイグレーションファイルをGitで共有することで、開発者全員が同じテーブルを作成することができる。  
  
### マイグレーションファイルのup()、down()は何のコマンドを実行した時に呼び出されるのか
up()	php artisan migrate を実行したとき  
down()	php artisan migrate:rollback を実行したとき  
  
### Seederクラスの役割は何か
データベースに初期データやテストデータを投入するためのクラス  
・テーブルに初期データを入れる  
例えば開発環境で Todo アプリを作るときに、サンプルの ToDo を登録しておく。  
  
・テスト用データの作成  
自動テストや画面確認のために、簡単にデータを生成できる  
  
・データベースの状態をリセットして再利用  
truncate() でテーブルを空にしてから再投入できる  
  
・複数の Seeder をまとめて実行  
DatabaseSeeder から $this->call() でまとめて呼び出せる  
  
### route関数の引数・返り値・使用するメリット
・引数  
第1引数：ルート名（web.php で ->name('todo.index') のように指定した名前）  
第2引数（任意）：ルートに渡すパラメータ（id など）  
    
・返り値  
指定したルートに対応する URL文字列 を返す  
    
・メリット  
ルートのURLをハードコーディングしなくてよい  
ルートの変更に強い（ルートのURLが変わっても blade側を書き換える必要なし）  
可読性が高く、パラメータ付きURLも安全に生成できる  
    
### @extends・@section・@yieldの関係性とbladeを分割するメリット
一覧画面と新規作成画面のBladeには、同じ記述が複数回登場するが、  
もし修正が必要な場合は全てのファイルの同じ記述を変更する必要があり、保守性が低い状態になる。  
そこで、Bladeの分割と継承を行います。これにより、共通の部分を別ファイルにまとめ、再利用可能にする。  
@extends('Bladeファイルのパス')を使用することで他のBladeファイルを継承することができる  
@section('任意の文字列') ~ @endsectionで囲った部分を継承したBladeファイルの@yield('任意の文字列')の箇所に挿入される。  
  
・メリット  
→共通レイアウトを再利用できる（ヘッダー・フッターを毎回書かなくてよい）  
→ビューを分割して管理しやすくなる  
→コードの可読性・保守性が向上  
    
### @csrfは何のための記述か
Bladeでフォームを作るときに CSRFトークンを自動で埋め込む。  
CSRF（Cross-Site Request Forgery）攻撃から守るため。  
  
### {{ }}とは何の省略系か
Blade の エスケープ出力のショートカット  
実際には echo e(...) と同じ意味。  
  
例：  
{{ $todo->content }}  
  
内部：  
<?php echo e($todo->content); ?>
